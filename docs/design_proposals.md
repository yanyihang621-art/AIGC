# 设计方案归档与模板 (Design Proposals)

本文件用于记录和规划针对“AI创意大赛 - 传统文化互动体验”应用的具体设计提案与交互草案。

---

## 提案模板 (Use this Template for New Proposals)

当您想发起一个新的交互方案设计时，可以复制下方格式并在本文件（或在 `docs/` 下新建单独的 markdown 文件）中进行细化：

```markdown
### [方案名称]：例如“水墨书法AI生成互动面板设计”

#### 1. 设计背景与目的
- **解决什么问题**：评委在扫码时第一眼需要感受什么？
- **预期达到的效果**：如何提供沉浸式的、有文化气息的反馈？

#### 2. 交互流程 (Interaction Flow)
- **触发状态 (Trigger)**：用户进行什么操作（如：点击按钮、手写输入）？
- **等待状态 (Pending)**：AI 生成时的过渡动效（水墨慢慢晕染开，呈现古诗字样的轮廓）？
- **呈现状态 (Render)**：生成完毕后的展示方式（淡入，伴随轻微的模糊滤镜退场）？

#### 3. 视觉表现与参数 (Visual Specs)
- **色彩策略**：是否遵循 `DESIGN.md` 的 `Restrained` 策略？
- **所选色值 (OKLCH)**：
  - 局部点缀是否使用了 **暗朱砂** `oklch(0.450 0.080 35)`？
- **字体与排版**：大字号使用衬线体（宋体），正文使用无衬线体。

#### 4. 评审检查 (Checklist)
- [ ] 按钮/链接的文案是否符合“动词+宾语”（例如“开始题字”而非“确定”）？
- [ ] 对比度是否合格（浅绢色背景上的黛墨文字是否足够清晰）？
- [ ] 移动端是否做好了防溢出处理？
```

### 提案 #001：新中式极简交互框架
*   **状态**：已在 `DESIGN.md` 中立项并作为全局基调。
