# 项目工作流说明 (Workflows)

本项目是一个结合了“新中式设计”与“AI互动”的传统文化创意应用。为了保证开发的高效以及设计风格的纯粹性，请遵循以下开发与协作流。

---

## 1. 运行与开发工作流

### 启动开发服务器
本地开发使用 Vite 作为打包和构建工具。
```bash
# 安装依赖（首次运行）
npm install

# 启动本地开发服务
npm run dev
```
启动后可在浏览器中访问 `http://localhost:5173/` 实时预览。

### 生产环境打包
在确保本地预览无误后，如需生成部署包，运行：
```bash
npm run build
```
打包产物将输出在 `dist/` 目录中。

---

## 2. 界面设计与 AI 协作流 (Impeccable)

本项目配置了 **Impeccable** 界面设计与审计框架。当您需要对页面、组件进行设计、改版或优化时，可以使用以下指令进行操作：

| 阶段 | 常用指令 | 说明 |
| :--- | :--- | :--- |
| **规划阶段** | `shape [功能名称]` | 在编写代码前，先规划 UX/UI 交互流程 |
| **开发实现** | `craft [功能名称]` | 从设计到代码实现端到端地完成一个新功能 |
| **设计审核** | `critique [目标文件]` | 让 AI 按照启发式评估标准对界面进行设计打分与评审 |
| **技术审计** | `audit [目标文件]` | 进行无障碍性（a11y）、性能（perf）和响应式适配的专项技术检查 |
| **细节打磨** | `polish [目标文件]` | 针对现有界面进行最终的高级细节打磨与视觉纠偏 |

> [!NOTE]
> 框架运行的核心依据是根目录下的 `PRODUCT.md`（产品定义）和 `DESIGN.md`（视觉设计规范）。每次设计迭代前，请确保这两个文件体现了最新想法。

---

## 3. 设计规范遵守与防范 (Anti-references)

在开发新交互和组件时，需时刻审查是否违背了 `DESIGN.md` 中确定的原则：
*   **不要过度设计**：保持大量的白色留白（宣纸感）。
*   **克制动效**：绝对不能使用生硬的弹簧动效（bounce），只能使用平滑的指数缓动（ease-out-expo），并使用模糊（blur）和渐变（opacity）模拟水墨效果。
*   **对比度合规**：文字与背景的对比度需符合 WCAG 标准（正文 ≥ 4.5:1）。
